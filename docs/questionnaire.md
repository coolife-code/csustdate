# 匹配问卷设计文档

## 概述

本文档定义 DateDrop 匹配系统的核心问卷设计。问卷收集用户的多维特征向量，用于计算匹配分数，是匹配算法的核心输入。问卷采用纯选择题形式，确保数据标准化和算法可计算性。

## 问卷数据结构

```
用户特征向量 (User Profile Vector)
├── 外貌与生活方式 (权重: 15%)
├── 性格特征 (权重: 20%)
├── 价值观与恋爱观 (权重: 25%)
├── 兴趣爱好 (权重: 15%)
├── 学业与规划 (权重: 10%)
├── 家庭与背景 (权重: 10%)
└── 期望偏好 (权重: 5%)
```

## 向量化规则

### 数值化方法

问卷答案需要转化为数值向量才能用于匹配计算。

#### 单选题数值化

单选题将选项字母映射为 0-1 范围的数值：

```
选项 A → 0.0
选项 B → 0.25
选项 C → 0.5
选项 D → 0.75
选项 E → 1.0
...
```

公式：`数值 = (选项索引) / (总选项数 - 1)`

示例：
- Q1.1 身高（8个选项）：A=0, B≈0.14, C≈0.29, D≈0.43, E≈0.57, F≈0.71, G≈0.86, H=1.0
- Q2.1 内外向（5个选项）：A=0, B=0.25, C=0.5, D=0.75, E=1.0

#### 多选题数值化

多选题计算选中选项的比例：

```
数值 = 选中选项数 / 总选项数
```

示例：
- Q4.1 电影类型（8个选项）：选中3个 → 3/8 = 0.375

## 匹配分数计算

### 相似度计算

两个用户在某个维度上的相似度：

```
相似度 = 1 - |用户A数值 - 用户B数值|
```

示例：
- 用户A的身高数值：0.57（170-175cm）
- 用户B的身高数值：0.71（175-180cm）
- 身高相似度 = 1 - |0.57 - 0.71| = 0.86

### 综合匹配分数

```
匹配分数 = Σ (维度相似度 × 维度权重) × 100
```

权重分配：
- 外貌与生活方式：15%
- 性格特征：20%
- 价值观与恋爱观：25%
- 兴趣爱好：15%
- 学业与规划：10%
- 家庭与背景：10%
- 期望偏好：5%

### 匹配理由生成

根据各维度相似度生成匹配理由：

| 相似度范围 | 匹配程度 |
|-----------|---------|
| ≥ 0.9 | 高度契合 |
| 0.7 - 0.9 | 比较契合 |
| 0.5 - 0.7 | 一般契合 |
| < 0.5 | 差异较大 |

系统会自动选取相似度最高的3个维度作为匹配理由展示给用户。
